What is Deadlock?

Deadlock is one of the most critical challenges in operating systems, occurring when processes become permanently blocked, waiting for resources that will never become available. In a multi-process environment where resources such as CPU cycles, memory, I/O devices, and file handles are shared, deadlock situations can bring system execution to a complete halt.

Reusable Resource Deadlock

+

Occurs when processes compete for resources that can be used repeatedly, such as processors, memory blocks, I/O devices, and data structures.

Example: Process A holds Resource 1 and requests Resource 2, while Process B holds Resource 2 and requests Resource 1, creating a circular wait condition.

Consumable Resource Deadlock

+

Arises with resources that are created and destroyed dynamically, such as messages, signals, and interrupts.

Example: Process A waits for a signal from Process B, and Process B waits for a signal from Process A, both remaining indefinitely blocked.

Static Deadlock

+

Occurs when resource requirements and allocation patterns are determined before execution begins. The deadlock is inherent in the program design.

Characteristic: Easier to detect and prevent through careful system design and static analysis.

Dynamic Deadlock

+

Develops during runtime based on the actual execution sequence and timing of resource requests.

Characteristic: More challenging to predict and handle because it depends on runtime conditions that may vary across executions.
